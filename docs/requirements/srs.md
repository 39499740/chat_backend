## 1. 引言

### 1.1 文档目的

本文档定义了微信类社交应用后端系统的完整功能和非功能需求。本文档作为：
- 软件架构设计的基础
- 系统开发和测试的依据
- 需求追踪和变更管理的参考
- 项目验收的标准

### 1.2 系统概述

本系统是一个企业级的社交应用后端系统，提供类似微信的功能，包括：
- 用户账户管理和身份认证
- 实时即时通讯（文字、语音、视频、表情、群聊）
- 好友关系管理
- 朋友圈（动态发布、评论、点赞）
- 文件上传和媒体存储
- 离线消息推送
- 系统通知
- 用户和内容搜索

### 1.3 范围和边界

#### 1.3.1 在范围内

- 用户注册、登录、资料管理
- 好友添加、删除、拉黑
- 私聊消息收发
- 群聊创建、管理和消息广播
- 语音、视频通话信令
- 朋友圈动态发布和浏览
- 评论和点赞功能
- 文件上传（图片、视频、音频）
- 系统通知推送
- 用户搜索

#### 1.3.2 不在范围内

- 前端客户端实现（Android/iOS/Web）
- 支付功能
- 小程序
- 公众号
- 视频转码服务器（使用第三方服务）
- 短信验证码服务（使用第三方服务）

## 2. 总体描述

### 2.1 产品概述

#### 2.1.1 产品愿景

构建一个高可扩展、高性能的社交应用后端系统，支持百万级用户和亿级消息，提供类似微信的完整社交体验。

#### 2.1.2 目标用户

- **主要用户**：18-45岁的社交网络用户
- **次要用户**：企业管理员、系统运维人员

#### 2.1.3 使用场景

1. 用户注册账号，填写个人资料
2. 搜索并添加好友
3. 创建或加入群聊
4. 发送文本、图片、语音、视频消息
5. 发起语音或视频通话
6. 发布朋友圈动态
7. 浏览好友动态、点赞和评论
8. 接收系统通知

### 2.2 用户特征

| 用户类型 | 技能水平 | 使用频率 | 关键需求 |
|---------|---------|---------|---------|
| 普通用户 | 中等 | 每天多次 | 快速响应、稳定连接、隐私保护 |
| 企业用户 | 中等到高等 | 每周多次 | 高可靠性、管理功能、数据安全 |
| 系统管理员 | 高 | 不定期 | 监控工具、日志、性能指标 |

### 2.3 约束条件

#### 2.3.1 技术约束

- 后端框架：NestJS (TypeScript)
- 数据库：MySQL 16+
- 缓存：Redis 7+
- 对象存储：MinIO (S3兼容)
- 实时通信：Socket.IO
- 容器化：Docker + Docker Compose

#### 2.3.2 性能约束

- API响应时间：P95 < 200ms
- WebSocket消息延迟：< 100ms
- 并发用户数：≥ 10,000
- 消息吞吐量：≥ 10,000 msg/s

#### 2.3.3 安全约束

- 所有API需要JWT认证
- 密码使用BCrypt加密（salt rounds: 10）
- 敏感数据传输使用HTTPS
- SQL注入防护
- XSS防护
- CSRF防护

#### 2.3.4 法规约束

- 用户数据隐私保护
- 数据存储本地化
- 审计日志保留90天

### 2.4 假设和依赖

#### 2.4.1 假设

- 用户使用现代浏览器或官方客户端
- 网络连接稳定
- 第三方服务（邮件、短信）可用

#### 2.4.2 依赖

- MySQL数据库服务
- Redis缓存服务
- MinIO对象存储服务
- 邮件发送服务（SMTP）
- 短信发送服务（可选）

## 3. 功能需求

### 3.1 用户管理 (USERS)

#### REQ-USERS-001: 用户注册

**描述**: 新用户可以通过手机号或邮箱注册账号。

**优先级**: 高

**输入**:
- 手机号或邮箱（唯一）
- 密码（8-20位，包含字母和数字）
- 用户名（3-32位，字母数字下划线）
- 昵称（可选）

**输出**:
- 用户ID (UUID)
- JWT Token
- Refresh Token

**验证规则**:
- 手机号格式验证
- 邮箱格式验证
- 密码强度验证
- 用户名唯一性验证
- 验证码验证（可选）

**异常处理**:
- 手机号/邮箱已存在
- 用户名已存在
- 密码强度不足
- 验证码无效或过期

#### REQ-USERS-002: 用户登录

**描述**: 注册用户可以使用手机号/邮箱或用户名登录。

**优先级**: 高

**输入**:
- 登录账号（手机号/邮箱/用户名）
- 密码

**输出**:
- 用户信息
- JWT Token (有效期7天)
- Refresh Token (有效期30天)
- 设备信息记录

**验证规则**:
- 账户格式验证
- 密码验证
- 账户状态验证（非禁用）

**异常处理**:
- 账户不存在
- 密码错误
- 账户被禁用
- 账户被锁定（5次失败后锁定30分钟）

#### REQ-USERS-003: 获取当前用户信息

**描述**: 已认证用户可以获取自己的账户信息。

**优先级**: 高

**前置条件**: 用户已登录（有效JWT Token）

**输出**:
- 用户ID
- 用户名
- 邮箱/手机号
- 昵称
- 头像URL
- 性别
- 生日
- 地区
- 注册时间
- 最后登录时间

#### REQ-USERS-004: 更新用户资料

**描述**: 用户可以更新个人资料信息。

**优先级**: 中

**前置条件**: 用户已登录

**输入**:
- 昵称
- 性别
- 生日
- 地区
- 个人简介
- 头像URL

**输出**: 更新后的用户信息

**限制**:
- 头像大小限制：5MB
- 头像格式：JPG, PNG
- 个人简介长度：≤ 200字

#### REQ-USERS-005: 修改密码

**描述**: 用户可以修改登录密码。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 原密码
- 新密码
- 确认新密码

**验证规则**:
- 原密码验证
- 新密码强度验证
- 新密码与确认密码一致性验证

**异常处理**:
- 原密码错误
- 新密码强度不足
- 新密码与原密码相同

#### REQ-USERS-006: 用户登出

**描述**: 用户可以主动登出系统。

**优先级**: 高

**前置条件**: 用户已登录

**功能**:
- 使JWT Token失效（黑名单或缩短有效期）
- 清除会话记录
- 更新在线状态为离线

### 3.2 好友系统 (FRIENDS)

#### REQ-FRIENDS-001: 发送好友请求

**描述**: 用户可以向其他用户发送好友请求。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 目标用户ID或手机号/邮箱
- 验证消息（可选，≤ 100字）

**验证规则**:
- 不能添加自己为好友
- 不能重复添加（已发送或已是好友）
- 目标用户存在

**输出**: 好友请求ID

**通知**: 目标用户收到好友请求通知

#### REQ-FRIENDS-002: 接受好友请求

**描述**: 用户可以接受收到的好友请求。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 好友请求ID

**功能**:
- 创建双向好友关系
- 删除好友请求记录
- 通知请求发送者

#### REQ-FRIENDS-003: 拒绝好友请求

**描述**: 用户可以拒绝收到的好友请求。

**优先级**: 中

**前置条件**: 用户已登录

**输入**:
- 好友请求ID

**功能**:
- 标记请求为已拒绝
- 通知请求发送者（可选）

#### REQ-FRIENDS-004: 获取好友列表

**描述**: 用户可以查看自己的好友列表。

**优先级**: 高

**前置条件**: 用户已登录

**输出**:
- 好友列表（分页）
- 每个好友的信息（ID、用户名、昵称、头像、备注）
- 在线状态

**排序**: 最近聊天时间、字母顺序

#### REQ-FRIENDS-005: 删除好友

**描述**: 用户可以删除好友关系。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 好友用户ID

**功能**:
- 删除双向好友关系
- 保留聊天记录（可选）
- 通知对方（可选）

#### REQ-FRIENDS-006: 拉黑用户

**描述**: 用户可以拉黑其他用户。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 目标用户ID

**功能**:
- 添加到黑名单
- 删除好友关系（如果存在）
- 屏蔽对方消息和动态
- 不能被搜索到

#### REQ-FRIENDS-007: 获取待处理好友请求

**描述**: 用户可以查看待处理的好友请求。

**优先级**: 高

**前置条件**: 用户已登录

**输出**:
- 待处理请求列表
- 每个请求的信息（请求者、消息、时间）

### 3.3 聊天系统 (CHAT)

#### REQ-CHAT-001: 发送私聊消息

**描述**: 用户可以向好友发送私聊消息。

**优先级**: 高

**前置条件**: 用户已登录，存在好友关系

**输入**:
- 接收者ID
- 消息类型（text/image/voice/video/emoji）
- 消息内容

**消息类型**:
- 文本：≤ 5000字
- 图片：≤ 10MB
- 语音：≤ 5MB，≤ 60秒
- 视频：≤ 50MB，≤ 5分钟

**输出**:
- 消息ID
- 时间戳
- 消息状态（发送中/已送达/已读）

**实时推送**: 通过WebSocket推送给接收者

#### REQ-CHAT-002: 获取私聊历史消息

**描述**: 用户可以查看与某人的聊天历史。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 对方用户ID
- 分页参数（before messageId, limit）
- 消息类型过滤（可选）

**输出**:
- 消息列表（按时间倒序）
- 消息内容、发送者、时间戳
- 分页信息

**限制**:
- 单次最多返回50条
- 仅返回双方好友关系期间的消息

#### REQ-CHAT-003: 标记消息已读

**描述**: 用户可以标记消息为已读。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 消息ID列表

**功能**:
- 更新消息已读状态
- 更新会话未读数
- 通知发送者（WebSocket）

#### REQ-CHAT-004: 撤回消息

**描述**: 用户可以在发送后2分钟内撤回消息。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 消息ID

**限制**:
- 仅能撤回自己发送的消息
- 发送后2分钟内可撤回

**功能**:
- 标记消息为已撤回
- 通知对方
- 保留撤回记录（审计）

#### REQ-CHAT-005: 创建群组

**描述**: 用户可以创建群组。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 群组名称（2-64字）
- 群组头像（可选）
- 初始成员ID列表（可选）
- 群组描述（可选，≤ 200字）

**限制**:
- 群成员上限：500人
- 群主自动成为管理员

**输出**:
- 群组ID
- 群组信息

**通知**: 所有初始成员收到加入群组通知

#### REQ-CHAT-006: 发送群聊消息

**描述**: 群组成员可以在群组中发送消息。

**优先级**: 高

**前置条件**: 用户已登录，是群组成员

**输入**:
- 群组ID
- 消息类型
- 消息内容

**功能**:
- 广播消息给所有在线成员
- 存储消息到数据库
- 离线消息队列处理

#### REQ-CHAT-007: 获取群组历史消息

**描述**: 群组成员可以查看群组历史消息。

**优先级**: 高

**前置条件**: 用户已登录，是群组成员

**输入**:
- 群组ID
- 分页参数

**输出**:
- 消息列表
- 发送者信息
- 分页信息

#### REQ-CHAT-008: 管理群组成员

**描述**: 群主和管理员可以管理群组成员。

**优先级**: 中

**前置条件**: 用户已登录，具有管理权限

**功能**:
- 添加成员
- 移除成员
- 设置/取消管理员
- 群主可以转让群主权限

#### REQ-CHAT-009: 离开群组

**描述**: 群组成员可以离开群组。

**优先级**: 中

**前置条件**: 用户已登录，是群组成员

**限制**:
- 群主不能离开（必须先转让权限）

#### REQ-CHAT-010: 视频通话信令

**描述**: 支持WebRTC视频/语音通话信令交换。

**优先级**: 中

**功能**:
- 发起通话
- 接受/拒绝通话
- 通话建立
- 通话结束
- ICE候选交换
- SDP交换

**服务器角色**:
- 信令服务器（中继信令消息）
- 提供STUN/TURN服务器信息

### 3.4 朋友圈 (MOMENTS)

#### REQ-MOMENTS-001: 发布动态

**描述**: 用户可以发布朋友圈动态。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 文字内容（可选，≤ 500字）
- 图片/视频（可选，最多9张图片或1个视频）
- 位置信息（可选）
- 可见性设置（公开/仅好友/私密/指定可见）

**限制**:
- 图片大小：≤ 10MB
- 视频大小：≤ 100MB，时长≤ 10分钟
- 发布频率：≤ 10条/小时

**功能**:
- 生成动态ID
- 处理媒体文件（压缩、缩略图）
- 推送到好友Feed

#### REQ-MOMENTS-002: 获取朋友圈动态列表

**描述**: 用户可以浏览朋友圈动态。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 分页参数
- 用户ID（可选，查看特定用户的动态）

**输出**:
- 动态列表
- 发布者信息
- 媒体URL
- 点赞数
- 评论数
- 是否已点赞

**可见性规则**:
- 公开：所有人可见
- 仅好友：仅好友可见
- 私密：仅自己可见
- 指定可见：仅指定用户可见

#### REQ-MOMENTS-003: 删除动态

**描述**: 用户可以删除自己的动态。

**优先级**: 中

**前置条件**: 用户已登录

**输入**:
- 动态ID

**功能**:
- 删除动态
- 删除相关评论和点赞
- 删除媒体文件（如果无其他引用）

#### REQ-MOMENTS-004: 点赞动态

**描述**: 用户可以点赞朋友的动态。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 动态ID

**功能**:
- 添加点赞记录
- 更新点赞计数
- 通知发布者

**限制**:
- 不能重复点赞
- 不能点赞自己的动态

#### REQ-MOMENTS-005: 取消点赞

**描述**: 用户可以取消点赞。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 动态ID

**功能**:
- 删除点赞记录
- 更新点赞计数

#### REQ-MOMENTS-006: 添加评论

**描述**: 用户可以对动态添加评论。

**优先级**: 高

**前置条件**: 用户已登录，可以查看该动态

**输入**:
- 动态ID
- 评论内容（≤ 200字）
- 回复评论ID（可选）

**功能**:
- 创建评论记录
- 更新评论计数
- 通知发布者和被回复者（如果有）

#### REQ-MOMENTS-007: 删除评论

**描述**: 用户可以删除自己的评论。

**优先级**: 中

**前置条件**: 用户已登录

**输入**:
- 评论ID

**功能**:
- 删除评论
- 更新评论计数
- 删除相关回复（可选）

### 3.5 通知系统 (NOTIFICATIONS)

#### REQ-NOTIF-001: 创建通知

**描述**: 系统在特定事件发生时创建通知。

**优先级**: 高

**通知类型**:
- 好友请求
- 好友请求被接受/拒绝
- 新消息
- 群组邀请
- 动态点赞
- 动态评论
- 评论回复

**功能**:
- 自动创建通知
- 推送到WebSocket（在线用户）
- 写入数据库

#### REQ-NOTIF-002: 获取通知列表

**描述**: 用户可以查看通知列表。

**优先级**: 高

**前置条件**: 用户已登录

**输入**:
- 分页参数
- 通知类型过滤（可选）
- 已读/未读过滤（可选）

**输出**:
- 通知列表
- 通知类型、内容、时间
